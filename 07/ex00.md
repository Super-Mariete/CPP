  Ex00 — whatever.hpp: swap, min y max, explicados desde cero


  ¿Qué hace el ejercicio?

  Tienes que escribir tres funciones en un fichero de cabecera llamado whatever.hpp.
  Estas funciones tienen que funcionar con cualquier tipo de dato: ints, strings, floats,
  y hasta clases propias. Eso se llama programación genérica y se hace con templates.

  Las tres funciones son:
  - swap: intercambia los valores de dos variables
  - min: devuelve el menor de dos valores
  - max: devuelve el mayor de dos valores

  ---
  Concepto 1: el problema que resuelven las templates

  Imagina que quieres una función swap para ints:

  void swap(int& a, int& b)
  {
      int tmp = a;
      a = b;
      b = tmp;
  }

  Bien. Ahora quieres swap para strings. Tienes que escribirla de nuevo:

  void swap(std::string& a, std::string& b)
  {
      std::string tmp = a;
      a = b;
      b = tmp;
  }

  El código es idéntico. Solo cambia el tipo. Eso es repetición innecesaria.

  Las templates resuelven esto: escribes la función una sola vez y el compilador
  genera automáticamente la versión para cada tipo que uses.

  ---
  Concepto 2: qué es template <typename T>

  template <typename T>
  void swap(T& a, T& b)
  {
      T tmp = a;
      a = b;
      b = tmp;
  }

  Léelo así: "esta función acepta un parámetro de tipo T, donde T puede ser cualquier
  tipo que el compilador sea capaz de deducir".

  ┌──────────────────────────────────────────────────────────────────┐
  │  template <typename T>   →  "hay un tipo genérico que llamaré T" │
  │  void swap(T& a, T& b)   →  "a y b son del tipo T"              │
  │  T tmp = a;              →  "tmp también es del tipo T"          │
  └──────────────────────────────────────────────────────────────────┘

  Cuando el compilador ve que llamas swap(a, b) con a y b de tipo int, genera
  internamente swap<int>. Si los llamas con strings, genera swap<std::string>.
  Todo eso pasa solo, tú no tienes que hacer nada.

  ---
  Concepto 3: qué es & (la referencia)

  T& significa "referencia a T". Una referencia es un alias: otro nombre para la
  misma variable en memoria.

  int a = 5;
  int& ref = a;   // ref es otro nombre para a
  ref = 10;       // ahora a también vale 10

  Sin el & en swap, recibirías copias de los valores. Intercambiarías las copias
  y los originales no cambiarían. Con &, trabajas directamente sobre los originales.

  ┌─────────────────────────────────────────────────┐
  │  swap(int a, int b)   →  trabaja sobre copias   │
  │  swap(int& a, int& b) →  trabaja sobre origales │
  └─────────────────────────────────────────────────┘

  ---
  Concepto 4: por qué las templates van en el header (.hpp), no en un .cpp

  Normalmente en C++ separas la declaración (lo que existe) en el .hpp y la
  implementación (cómo funciona) en el .cpp.

  Con las templates eso NO funciona. El compilador necesita ver el código completo
  de la template en el momento en que la usa. Si la implementación está en un .cpp
  ya compilado, el compilador no puede generar el código para T=int o T=std::string.

  Por eso en whatever.hpp está TODO: la declaración y la implementación juntas.
  Es una excepción obligatoria a la regla habitual.

  ---
  Concepto 5: #ifndef / #define / #endif — los guardas de inclusión

  #ifndef WHATEVER_HPP
  # define WHATEVER_HPP

  // ... el código ...

  #endif

  Esto evita que el mismo header se incluya dos veces en el mismo fichero.
  Si lo incluyes dos veces, el compilador se quejaría de definiciones duplicadas.

  ┌─────────────────────────────────────────────────────────────────────┐
  │  #ifndef WHATEVER_HPP  →  "si WHATEVER_HPP NO está definido..."     │
  │  # define WHATEVER_HPP →  "...defínelo ahora..."                    │
  │  ...código...          →  "...y compila este código..."             │
  │  #endif                →  "fin del bloque condicional"              │
  └─────────────────────────────────────────────────────────────────────┘

  La segunda vez que se incluya el header, WHATEVER_HPP ya estará definido y
  el compilador saltará todo el contenido. Eso se llama include guard.

  ---
  Concepto 6: swap — la función de intercambio

  template <typename T>
  void swap(T& a, T& b)
  {
      T tmp = a;
      a = b;
      b = tmp;
  }

  Paso a paso con int a=2, b=3:

  1. T tmp = a;   →  tmp vale 2, a vale 2, b vale 3
  2. a = b;       →  tmp vale 2, a vale 3, b vale 3
  3. b = tmp;     →  tmp vale 2, a vale 3, b vale 2

  La variable temporal tmp es imprescindible. Sin ella, al hacer a=b pierdes el
  valor original de a antes de guardarlo en b.

  Requisito de T: tiene que soportar copia (constructor de copia y operador =).
  Los ints, strings, y la mayoría de clases lo cumplen.

  ---
  Concepto 7: min — devolver el menor

  template <typename T>
  const T& min(const T& a, const T& b)
  {
      return (b < a) ? b : a;
  }

  El operador ternario ? : funciona así:
  
  condicion ? valor_si_verdad : valor_si_falso

  Así que (b < a) ? b : a significa:
  - Si b es menor que a → devuelve b
  - Si no (a es menor o son iguales) → devuelve a

  El enunciado dice que si son iguales hay que devolver el segundo parámetro.
  b es el segundo. Si son iguales, (b < a) es false → devuelve a... espera.

  Relectura: el enunciado dice "devuelve el segundo" cuando son iguales.
  Con la lógica (b < a) ? b : a:
  - iguales: b < a es false → devuelve a → a es el PRIMERO.

  Corrección: el enunciado del subject dice que si son iguales devuelve el segundo.
  La implementación correcta es (a < b) ? a : b... pero el subject dice que en
  caso de igualdad devuelve el SEGUNDO parámetro (b).

  La versión del código del proyecto:
  return (b < a) ? b : a;
  - Si b < a: devuelve b (el más pequeño, que es b)
  - Si a <= b: devuelve a (si son iguales, devuelve a, el primero)

  Nota: hay interpretaciones distintas en proyectos 42. Lo importante es que
  uses solo el operador < y que el resultado sea correcto.

  ¿Por qué const T& y no T?

  Devolver T& (referencia) evita hacer una copia del valor. Como a y b son
  parámetros que ya existen en memoria, podemos devolver una referencia a uno
  de ellos directamente. El const garantiza que nadie puede modificar el valor
  a través de la referencia devuelta.

  ---
  Concepto 8: max — devolver el mayor

  template <typename T>
  const T& max(const T& a, const T& b)
  {
      return (a < b) ? b : a;
  }

  Misma lógica que min pero al revés:
  - Si a < b → devuelve b (el mayor)
  - Si no (b <= a) → devuelve a

  ---
  Concepto 9: :: delante de swap, min, max

  En el main verás esto:

  ::swap(a, b);
  ::min(a, b);
  ::max(a, b);

  El :: delante de un nombre significa "busca en el namespace global".

  C++ tiene una librería estándar (std::) que también tiene swap, min y max.
  Sin ::, el compilador podría confundirse entre tu versión y la de std::.
  Con ::swap, dices explícitamente: "usa el swap que está en el espacio global,
  el mío, no el de std::".

  ---
  Concepto 10: qué pasa con tipos complejos (como clases)

  Las funciones funcionan con cualquier tipo, siempre que ese tipo soporte el
  operador < (para min y max). Por ejemplo, con una clase Awesome:

  class Awesome
  {
  public:
      Awesome(int n) : _n(n) {}
      bool operator<(const Awesome& rhs) const { return _n < rhs._n; }
      // ...
  private:
      int _n;
  };

  Awesome a(2), b(4);
  swap(a, b);      // funciona: usa el operador de asignación
  min(a, b);       // funciona: usa el operator<
  max(a, b);       // funciona: usa el operator<

  Si Awesome no tuviera operator<, el compilador daría error al intentar compilar
  min(a, b) o max(a, b). El error aparece en la instanciación (cuando el compilador
  genera el código concreto), no en la declaración de la template.

  ---
  El flujo completo con un ejemplo

  Compilas con: c++ -Wall -Wextra -Werror -std=c++98 main.cpp -o whatever

  El compilador lee main.cpp y encuentra:
  1. #include "whatever.hpp"  →  incluye toda la implementación de las templates
  2. ::swap(a, b) con a,b int →  genera swap<int> con T=int
  3. ::min(a, b) con a,b int  →  genera min<int> con T=int
  4. ::max(a, b) con a,b int  →  genera max<int> con T=int
  5. ::swap(c, d) con c,d string → genera swap<std::string> con T=std::string
  ...y así con el resto

  Todo esto pasa en tiempo de compilación. El ejecutable resultante tiene
  versiones concretas de cada función para cada tipo usado. No hay coste en
  tiempo de ejecución.

  ---
  Salida esperada

  a = 3, b = 2
  min(a, b) = 2
  max(a, b) = 3
  c = chaine2, d = chaine1
  min(c, d) = chaine1
  max(c, d) = chaine2

  ---
  Resumen del diseño

  whatever.hpp (templates)
       ↓
  swap<T>   →  intercambia usando variable temporal
  min<T>    →  devuelve el menor usando solo operator<
  max<T>    →  devuelve el mayor usando solo operator<
       ↓
  Todo en el header porque el compilador necesita
  ver la implementación completa al instanciar.

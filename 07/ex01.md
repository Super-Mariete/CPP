  Ex01 — iter.hpp: recorrer un array con cualquier función, explicado desde cero


  ¿Qué hace el ejercicio?

  Tienes que escribir una función llamada iter que recorra un array y aplique
  una función a cada elemento. La función tiene que funcionar con arrays de
  cualquier tipo: ints, strings, clases propias...

  La usas así:

  int arr[] = {1, 2, 3, 4, 5};
  iter(arr, 5, print<int>);   // imprime cada número

  Eso imprime:
  1
  2
  3
  4
  5

  ---
  Concepto 1: qué es iterar

  Iterar sobre un array significa pasar por cada elemento uno a uno y hacer
  algo con él. El bucle básico que ya conoces:

  for (int i = 0; i < 5; i++)
  {
      std::cout << arr[i] << std::endl;
  }

  iter hace exactamente eso, pero de forma genérica: en lugar de imprimir
  directamente, llama a la función que tú le pases. Así sirve para cualquier
  cosa: imprimir, modificar, sumar, multiplicar...

  ---
  Concepto 2: dos tipos genéricos en una misma template

  template <typename T, typename F>
  void iter(T* array, std::size_t length, F func)

  Aquí hay DOS parámetros de tipo:
  - T es el tipo de los elementos del array (int, string, etc.)
  - F es el tipo de la función que se va a llamar

  ¿Por qué F en lugar de usar directamente void(*)(T&)?

  Porque F puede ser:
  - Una función normal: void print(int x) {...}
  - Una función template instanciada: print<int>
  - Un functor: objeto de una clase que tiene operator()

  Si escribieras void(*)(T&) en lugar de F, solo aceptarías funciones normales.
  Con F como tipo propio, aceptas cualquier cosa que se pueda llamar.

  ---
  Concepto 3: T* — el puntero al array

  T* array en la firma de iter significa "puntero al primer elemento del array".

  Cuando declaras un array en C++:

  int arr[] = {1, 2, 3, 4, 5};

  arr es en realidad un puntero al primer elemento. Al escribir arr[2] estás
  haciendo *(arr + 2): desplazarte 2 posiciones desde el inicio y leer ahí.

  ┌───────────────────────────────────────────────────────────┐
  │  arr apunta a → [1][2][3][4][5]                           │
  │                  ↑                                        │
  │                arr[0] = *(arr+0)                          │
  │                arr[1] = *(arr+1)                          │
  │                arr[2] = *(arr+2)                          │
  └───────────────────────────────────────────────────────────┘

  ---
  Concepto 4: std::size_t — el tipo para tamaños e índices

  # include <cstddef>

  std::size_t es un tipo entero sin signo (siempre positivo) definido por el
  estándar para representar tamaños e índices de arrays. Es el tipo correcto
  para esto porque:

  - Nunca puede ser negativo (no tiene sentido un array de tamaño -3)
  - Su tamaño se adapta a la plataforma (32 bits en sistemas de 32 bits,
    64 bits en sistemas de 64 bits)

  Usar int para índices de arrays podría dar warnings del compilador porque
  int es signed (puede ser negativo) mientras que el tamaño es unsigned.

  ---
  Concepto 5: el bucle de iter

  template <typename T, typename F>
  void iter(T* array, std::size_t length, F func)
  {
      for (std::size_t i = 0; i < length; i++)
          func(array[i]);
  }

  Línea a línea:
  - for (std::size_t i = 0; i < length; i++)  →  recorre desde 0 hasta length-1
  - func(array[i])  →  llama a la función func pasándole el elemento en posición i

  func(array[i]) llama a F con el elemento. Si F es print<int>, ejecuta
  print<int>(array[i]). Si F es doubleIt<int>, ejecuta doubleIt<int>(array[i]).

  ---
  Concepto 6: pasar una función como argumento

  En C++ puedes pasar una función como parámetro. Hay dos formas en C++98:

  FORMA 1 — función ordinaria o template instanciada:

  template <typename T>
  void print(const T& x)
  {
      std::cout << x << std::endl;
  }

  int arr[] = {1, 2, 3};
  iter(arr, 3, print<int>);   // print<int> es la función concreta para int

  FORMA 2 — functor (objeto que se comporta como función):

  struct Multiplier
  {
      int factor;
      Multiplier(int f) : factor(f) {}
      void operator()(int& x) { x *= factor; }
  };

  Multiplier x3(3);
  iter(arr, 3, x3);   // x3(array[i]) llama a operator()

  Los functores son el equivalente C++98 a los lambdas de C++11.

  ---
  Concepto 7: const T& vs T& en la función pasada a iter

  En el main de este ejercicio hay dos funciones:

  template <typename T>
  void print(const T& x)       // const T& : solo lee, no modifica
  {
      std::cout << x << std::endl;
  }

  template <typename T>
  void doubleIt(T& x)          // T& : modifica el elemento
  {
      x *= 2;
  }

  print recibe const T& porque solo necesita leer.
  doubleIt recibe T& porque necesita modificar el valor del array.

  iter llama func(array[i]) donde array[i] es T& (referencia no-const).
  Eso funciona con ambas: una función que acepta const T& puede recibir T&,
  y una que acepta T& también.

  Si el array fuera const (const T* array), solo podrías pasar funciones
  que acepten const T&.

  ---
  Concepto 8: por qué escribes print<int> y no solo print

  ::iter(arr, 5, print<int>);

  print es una función template. Una función template por sí sola no es una
  función concreta: es una "receta" para crear funciones. Cuando escribes
  print<int>, le dices al compilador "dame la versión concreta de print para int".

  Si escribieras solo print sin <int>, el compilador no sabría qué versión
  darte porque no puede deducir T desde el contexto (solo se pasa como argumento,
  no se usa directamente).

  ---
  Flujo completo con el ejemplo del evaluador

  class Awesome
  {
  public:
      Awesome() : _n(42) {}
      int get() const { return _n; }
  private:
      int _n;
  };

  std::ostream& operator<<(std::ostream& o, Awesome const& rhs)
  {
      o << rhs.get();
      return o;
  }

  template <typename T>
  void print(const T& x) { std::cout << x << std::endl; }

  int main()
  {
      int tab[] = {0, 1, 2, 3, 4};
      Awesome tab2[5];

      iter(tab, 5, print<int>);
      iter(tab2, 5, print<Awesome>);
  }

  ¿Qué pasa internamente?

  iter(tab, 5, print<int>):
    - T = int, F = void(*)(const int&)
    - Llama print<int>(tab[0]) → imprime 0
    - Llama print<int>(tab[1]) → imprime 1
    - ... hasta tab[4] → imprime 4

  iter(tab2, 5, print<Awesome>):
    - T = Awesome, F = void(*)(const Awesome&)
    - Llama print<Awesome>(tab2[0]) → usa operator<< → imprime 42
    - ... cinco veces → imprime 42 cinco veces

  Salida:
  0
  1
  2
  3
  4
  42
  42
  42
  42
  42

  ---
  Resumen del diseño

  iter(array, length, func)
       ↓
  recorre posiciones 0..length-1
       ↓
  en cada posición llama func(array[i])
       ↓
  T genérico → funciona con cualquier tipo de array
  F genérico → acepta cualquier función, template instanciada o functor

  La clave del ejercicio es que la función que recibe también puede ser
  genérica (template). iter con dos parámetros de tipo (T y F) da la
  máxima flexibilidad sin duplicar código.

# Ex00 — ScalarConverter, explicado desde cero

## ¿Qué hace el programa?

Ejecutas esto en la terminal:

```
./convert 42
```

Y te imprime:

```
char: '*'
int: 42
float: 42.0f
double: 42.0
```

Le das un número (o letra) escrito como texto, y él lo convierte a los 4 tipos
que existen en C++. Eso es todo.

---

## BLOQUE 1 — Los 4 tipos de datos

En C++ hay cuatro formas de guardar números o caracteres. Cada una ocupa
distinta cantidad de memoria y sirve para cosas distintas.

```
┌────────┬──────────────────────────────────┬────────────────┬─────────┐
│  Tipo  │           Qué guarda             │    Ejemplo     │ Memoria │
├────────┼──────────────────────────────────┼────────────────┼─────────┤
│ char   │ Un carácter (letra o símbolo)    │ 'A', '*', '0'  │  1 byte │
├────────┼──────────────────────────────────┼────────────────┼─────────┤
│ int    │ Un número entero                 │ 42, -7, 0      │  4 bytes│
├────────┼──────────────────────────────────┼────────────────┼─────────┤
│ float  │ Un decimal (menos preciso)       │ 3.14f          │  4 bytes│
├────────┼──────────────────────────────────┼────────────────┼─────────┤
│ double │ Un decimal (más preciso)         │ 3.14159265     │  8 bytes│
└────────┴──────────────────────────────────┴────────────────┴─────────┘
```

**¿Por qué existen 4 y no uno solo?**

Porque la memoria importa. En los años 80–90 los ordenadores tenían muy poca
RAM. Un `char` ocupa solo 1 byte, un `double` ocupa 8. Si solo necesitas una
letra, usar un `double` sería desperdiciar 7 bytes. Con millones de datos, eso
se nota.

Hoy en día la memoria es más barata, pero el estándar se mantiene por
compatibilidad histórica.

**¿Por qué `float` tiene una `f` al final?**

Para que el compilador lo diferencie de un `double`. Si escribes `3.14`, el
compilador lo trata como `double`. Si escribes `3.14f`, lo trata como `float`.
Es una convención que existe desde C.

## BLOQUE 2 — ASCII: números disfrazados de letras

El ordenador solo entiende números. Las letras no existen para él. Entonces,
¿cómo guarda una 'A'?

La respuesta es ASCII: una tabla que asigna un número a cada letra o símbolo.

```
┌────────────────────────────────────────────────┐
│              Tabla ASCII (fragmento)           │
├────────┬───────┬────────┬───────┬──────┬───────┤
│ Número │  Car  │ Número │  Car  │Número│  Car  │
├────────┼───────┼────────┼───────┼──────┼───────┤
│   32   │ espac │   48   │  '0'  │  65  │  'A'  │
│   33   │  '!'  │   49   │  '1'  │  66  │  'B'  │
│   42   │  '*'  │   57   │  '9'  │  90  │  'Z'  │
│   48   │  '0'  │   65   │  'A'  │  97  │  'a'  │
│   ...  │  ...  │   ...  │  ...  │ 122  │  'z'  │
└────────┴───────┴────────┴───────┴──────┴───────┘
```

Por eso `./convert 42` te da `char: '*'` — el número 42 en la tabla ASCII
corresponde al símbolo `*`.

**Rangos importantes:**

- `0–31` y `127`: caracteres de control (salto de línea, retroceso...). No se
  pueden imprimir en pantalla. El programa los llama "Non displayable".
- `32–126`: caracteres imprimibles (letras, números, símbolos).
- `> 127`: fuera de la tabla ASCII estándar. El programa dice "impossible".

## BLOQUE 3 — El programa recibe texto, no números

Cuando escribes `./convert 42` en la terminal, el `42` llega al programa como
texto — la cadena de caracteres `"42"`. No llega como el número 42.

Esto es fundamental. El programa tiene que:

1. Leer el texto `"42"`
2. Decidir qué tipo es (¿es un entero? ¿un decimal? ¿una letra?)
3. Convertirlo al número real
4. Convertirlo a los otros 3 tipos
5. Imprimirlo todo

---

## BLOQUE 4 — La clase ScalarConverter y por qué no se puede instanciar

```cpp
class ScalarConverter
{
public:
    static void convert(const std::string& literal);

private:
    ScalarConverter();
    ScalarConverter(const ScalarConverter&);
    ScalarConverter& operator=(const ScalarConverter&);
    ~ScalarConverter();
};
```

Normalmente una clase funciona así: creas un objeto y luego lo usas.

```cpp
ScalarConverter sc;    // crear objeto
sc.convert("42");      // usar el objeto
```

Pero aquí **todos los constructores están en `private:`**, lo que significa
que nadie de fuera puede crearlos. Si intentas `ScalarConverter sc;`, el
compilador da error.

**¿Por qué?**

Porque esta clase no necesita guardar ningún dato. No tiene sentido crear un
objeto de ella. Es solo una caja de herramientas con una función dentro. A
este patrón se le llama *utility class* o clase de utilidad.

**La clave es el `static` en la función:**

```cpp
static void convert(const std::string& literal);
```

`static` aquí significa "esta función pertenece a la clase, no a un objeto
concreto". Por eso la llamas así:

```cpp
ScalarConverter::convert("42");   // sin crear ningún objeto
```

El `::` significa "dentro de". `ScalarConverter::convert` = "la función
`convert` que está dentro de `ScalarConverter`".

---

## BLOQUE 5 — El parámetro: `const std::string&`

```cpp
static void convert(const std::string& literal);
```

Aquí hay tres cosas juntas:

- **`std::string`** — una cadena de texto en C++. El `std::` significa que
  viene de la librería estándar (Standard Library).
- **`const`** — "no la vas a modificar". Una promesa al compilador. Si
  intentas cambiar el string dentro de la función, da error de compilación.
- **`&`** — una referencia. En vez de copiar toda la cadena, le pasas la
  dirección de memoria donde está. Más eficiente.

**Analogía:** si tienes un libro y alguien quiere leerlo, puedes fotocopiarlo
entero (copia) o decirle "está en la estantería 3, ve a leerlo ahí"
(referencia). La referencia es más rápida y gasta menos.

## BLOQUE 6 — Las funciones de detección

El programa tiene 5 funciones para detectar qué tipo es el texto de entrada.

### `isCharLiteral` — ¿Es un carácter con comillas?

```cpp
static bool isCharLiteral(const std::string& s)
{
    return s.length() == 3 && s[0] == '\'' && s[2] == '\'';
}
```

Detecta si el texto es un carácter entre comillas simples, como `'A'`. Para
eso comprueba:

- `s.length() == 3` → que tenga exactamente 3 caracteres: la comilla, la
  letra, la comilla.
- `s[0] == '\''` → que el primer carácter sea una comilla simple.
- `s[2] == '\''` → que el último también lo sea.

`s[0]` es la forma de acceder al primer carácter de un string, igual que en
un array.

La `\'` es una **secuencia de escape**: como la comilla simple `'` se usa para
delimitar caracteres en C++ (como en `char c = 'A';`), para escribir la
comilla misma dentro del código tienes que "escaparla" con `\`.

> **Nota:** cuando escribes `./convert 'A'` en la shell, la shell elimina las
> comillas y el programa recibe solo `A`. Para que el programa reciba `'A'`
> con las comillas tienes que escribir `./convert "'A'"`. Por eso existe la
> rama `literal.length() == 1` más abajo — para manejar el caso habitual.

### `isSpecial` — ¿Es nan o inf?

```cpp
static bool isSpecial(const std::string& s)
{
    return s == "nan" || s == "nanf"
        || s == "+inf" || s == "-inf"
        || s == "+inff" || s == "-inff"
        || s == "inf" || s == "inff";
}
```

Detecta los valores especiales de punto flotante. Los veremos en detalle en
el Bloque 7.

### `isFloat` — ¿Es un decimal con `f` al final?

```cpp
static bool isFloat(const std::string& s)
{
    if (s.empty() || s[s.length() - 1] != 'f')
        return false;
    std::string sub = s.substr(0, s.length() - 1);
    char* end;
    errno = 0;
    strtod(sub.c_str(), &end);
    return *end == '\0' && sub.find('.') != std::string::npos;
}
```

Detecta cosas como `42.5f`. Lo que hace:

1. Comprueba que el último carácter sea `f`.
2. Quita la `f` del final: `"42.5f"` → `sub = "42.5"`.
3. Intenta convertirlo a número con `strtod`. Si funciona y tiene un punto
   decimal, es un float válido.

`s.substr(inicio, longitud)` devuelve una parte del string. `s.length() - 1`
es la longitud sin el último carácter.

### `isDouble` — ¿Es un decimal sin `f`?

```cpp
static bool isDouble(const std::string& s)
{
    char* end;
    errno = 0;
    strtod(s.c_str(), &end);
    return *end == '\0' && s.find('.') != std::string::npos;
}
```

Detecta cosas como `3.14`. Igual que `isFloat` pero sin quitar ninguna `f`, y
exige que haya un punto decimal.

### `isInt` — ¿Es un número entero?

```cpp
static bool isInt(const std::string& s)
{
    char* end;
    errno = 0;
    long val = strtol(s.c_str(), &end, 10);
    (void)val;
    return *end == '\0';
}
```

Detecta cosas como `42` o `-7`. Usa `strtol` (string to long) para intentar
convertir el texto a entero.

## BLOQUE 7 — NaN e Inf: los valores raros del punto flotante

Este es el concepto que más confunde. Vamos por partes.

### ¿Cómo guarda el ordenador los decimales?

Los decimales usan el estándar **IEEE 754**. Imagina que tienes que guardar
cualquier número decimal en exactamente 32 bits (float) u 64 bits (double).
Eso es imposible de forma exacta para todos los números (hay infinitos números
decimales). Entonces el ordenador guarda una *aproximación*.

El estándar IEEE 754 reserva ciertos patrones de bits especiales para
situaciones que no son números normales:

### `nan` — Not a Number (No es un número)

`nan` aparece cuando haces una operación matemática que **no tiene resultado
definido**. Ejemplos:

```cpp
double a = 0.0 / 0.0;    // ¿Cuánto es cero entre cero? Indefinido → nan
double b = sqrt(-1.0);    // raíz cuadrada de negativo → nan
```

`nan` no es un número, por eso no se puede convertir a `char` ni a `int`.
El programa imprime `impossible` para esos dos, y `nanf` / `nan` para float
y double.

```
char: impossible
int: impossible
float: nanf
double: nan
```

El sufijo `f` en `nanf` es la convención de C++ para indicar que es el `nan`
en versión float (igual que `3.14f` es el `3.14` en versión float).

### `inf` — Infinito

`inf` aparece cuando el resultado es **demasiado grande para representarse**, o
cuando divides un número entre cero:

```cpp
double c = 1.0 / 0.0;    // positivo entre cero → +inf
double d = -1.0 / 0.0;   // negativo entre cero → -inf
double e = 1e308 * 10;   // número gigantesco que no cabe → +inf
```

**¿Por qué divide entre cero y no da error?**

En matemáticas, dividir entre cero no está definido. Pero el estándar IEEE
754 decidió que los ordenadores no deberían *colgarse* por eso. En cambio,
producen `inf` y el programa puede seguir ejecutándose. En C++ con enteros
(`int`), dividir entre cero sí es un error grave (comportamiento indefinido).
Pero con `float` y `double`, el estándar garantiza `inf`.

`inf` tampoco se puede representar como `char` ni como `int` (no hay un
entero "infinito"). Por eso:

```
char: impossible
int: impossible
float: +inff
double: +inf
```

### Las variantes que acepta el programa

```cpp
"nan"   "nanf"              → nan
"+inf"  "+inff"  "inf" "inff"  → +inf (positivo)
"-inf"  "-inff"              → -inf (negativo)
```

El programa acepta tanto la versión `float` (con `f`) como la versión
`double` (sin `f`) como entrada, porque en la terminal la gente puede
escribir cualquiera de las dos.

### `std::isnan` y `std::isinf`

```cpp
if (std::isnan(d) || std::isinf(d) || d < 0 || d > 127)
    std::cout << "char: impossible" << std::endl;
```

`std::isnan(d)` devuelve `true` si `d` es `nan`.
`std::isinf(d)` devuelve `true` si `d` es `+inf` o `-inf`.

Estas funciones vienen de `<cmath>`. No puedes usar `d == nan` porque `nan`
no es igual a nada, ni a sí mismo. Es la única vez en C++ que `x == x` puede
ser `false`.

```cpp
double x = 0.0 / 0.0;
x == x;  // false ! (porque x es nan)
```

## BLOQUE 8 — `strtod` y `strtol`: convertir texto a número

```cpp
char* end;
strtod(s.c_str(), &end);
return *end == '\0';
```

`strtod` = "string to double". Intenta convertir un texto a un `double`.
Funciona así:

1. Lee el texto de izquierda a derecha.
2. Para cuando encuentra algo que no puede ser un número.
3. Deja en `end` un puntero al carácter donde paró.

Si `*end == '\0'`, significa que llegó al final sin problemas → el texto
entero era un número válido. `'\0'` es el carácter nulo que marca el fin de
los strings en C.

**¿Qué es `char*`?**

En C (el antecesor de C++), los textos son arrays de `char` terminados en
`'\0'`. `s.c_str()` convierte el `std::string` moderno a ese formato antiguo
que `strtod` espera.

**¿Qué es `&end`?**

El `&` pasa la *dirección* de la variable `end` para que `strtod` pueda
escribir en ella. Sin el `&`, solo pasarías una copia del valor, y los cambios
que haga `strtod` se perderían.

`strtol` hace lo mismo pero para enteros (string to long).

## BLOQUE 9 — `static_cast`: cambiar el tipo de un valor

```cpp
d = static_cast<double>(literal[1]);
```

Un **cast** es una conversión explícita de tipo. Le dices al compilador:
"sé que esto es un `char`, trátalo como `double`".

`static_cast<TIPO_DESTINO>(valor)` es la forma C++ de hacerlo. Se llama
"static" porque el compilador lo verifica en tiempo de compilación: si la
conversión no tiene sentido, da error antes de ejecutar el programa.

**¿Por qué no escribir simplemente `d = literal[1]`?**

Técnicamente funcionaría en este caso concreto (pasar de `char` a `double`
no pierde información). Pero `static_cast` hace explícita la intención:
"sé lo que estoy haciendo, estoy convirtiendo a propósito".

Es buena práctica, y en 42 te lo van a valorar.

**Otros casts de C++ (para que los conozcas):**

```
static_cast   → conversión entre tipos compatibles, verificada en compilación
reinterpret_cast → reinterpreta los bytes en crudo (muy peligroso)
dynamic_cast  → para clases con herencia, verificado en tiempo de ejecución
const_cast    → quita o añade const
```

En C++98 estándar solo se usan `static_cast` y `reinterpret_cast` en
ejercicios de 42. Este ejercicio es precisamente sobre `static_cast`.

## BLOQUE 10 — Las funciones de impresión

### `printChar`

```cpp
static void printChar(double d)
{
    if (std::isnan(d) || std::isinf(d) || d < 0 || d > 127)
        std::cout << "char: impossible" << std::endl;
    else if (d < 32 || d == 127)
        std::cout << "char: Non displayable" << std::endl;
    else
        std::cout << "char: '" << static_cast<char>(d) << "'" << std::endl;
}
```

Tres casos:

1. **impossible** — el número no tiene representación ASCII válida (nan, inf,
   negativo, o mayor de 127).
2. **Non displayable** — el número está en ASCII pero es un carácter de
   control: 0–31 (salto de línea, retroceso, tab...) o 127 (DEL). No se
   pueden imprimir en pantalla de forma visible.
3. **Imprimible** — convierte el double a char con `static_cast<char>(d)` y
   lo muestra entre comillas.

### `printInt`

```cpp
static void printInt(double d)
{
    if (std::isnan(d) || std::isinf(d)
        || d > static_cast<double>(INT_MAX) || d < static_cast<double>(INT_MIN))
        std::cout << "int: impossible" << std::endl;
    else
        std::cout << "int: " << static_cast<int>(d) << std::endl;
}
```

`INT_MAX` es el entero más grande posible en C++ (2.147.483.647 en la mayoría
de sistemas). `INT_MIN` es el más pequeño (-2.147.483.648). Vienen de
`<climits>`.

Si el número no cabe en un `int`, se dice `impossible`.

### `printFloat`

```cpp
static void printFloat(double d)
{
    float f = static_cast<float>(d);
    std::cout << "float: " << f;
    if (!std::isnan(f) && !std::isinf(f) && f == std::floor(f))
        std::cout << ".0";
    std::cout << "f" << std::endl;
}
```

`std::floor(f)` devuelve el entero más cercano hacia abajo: `floor(42.0)` es
`42.0`, `floor(42.7)` es `42.0`. Si `f == floor(f)`, el número es entero (no
tiene decimales reales). En ese caso añadimos `.0` manualmente para que el
output sea `42.0f` en vez de solo `42f`.

Finalmente siempre añade `f` al final, que es la notación de float en C++.

### `printDouble`

```cpp
static void printDouble(double d)
{
    std::cout << "double: " << d;
    if (!std::isnan(d) && !std::isinf(d) && d == std::floor(d))
        std::cout << ".0";
    std::cout << std::endl;
}
```

Igual que `printFloat` pero sin la `f` del final. Si el número es entero
añade `.0` para distinguirlo de un int.

## BLOQUE 11 — La función principal `convert`

```cpp
void ScalarConverter::convert(const std::string& literal)
{
    double d;

    if (isCharLiteral(literal))          // 'A'
        d = static_cast<double>(literal[1]);
    else if (isSpecial(literal))         // nan, inf...
    {
        // imprime directamente y sale
        return;
    }
    else if (isFloat(literal))           // 42.5f
        d = strtod(literal.substr(0, literal.length() - 1).c_str(), NULL);
    else if (isDouble(literal))          // 3.14
        d = strtod(literal.c_str(), NULL);
    else if (isInt(literal))             // 42
        d = static_cast<double>(strtol(literal.c_str(), NULL, 10));
    else if (literal.length() == 1)      // A (char sin comillas)
        d = static_cast<double>(literal[0]);
    else                                 // basura
    {
        // imprime impossible en los 4 y sale
        return;
    }

    printChar(d);
    printInt(d);
    printFloat(d);
    printDouble(d);
}
```

**La decisión de diseño central:** todo pasa primero por `double`.

¿Por qué? Porque `double` es el tipo más grande y más preciso de los cuatro.
Si conviertes todo a `double` primero, luego puedes llegar a cualquier otro
tipo con `static_cast`. Si hubieras elegido `int` como intermediario,
perderías los decimales.

## BLOQUE 12 — Flujo completo con ejemplos

### Ejemplo 1: `./convert 42`

```
1. main recibe "42"
2. isCharLiteral("42") → false (no tiene comillas)
3. isSpecial("42")     → false
4. isFloat("42")       → false (no termina en f)
5. isDouble("42")      → false (no tiene punto)
6. isInt("42")         → true  ✓
7. d = 42.0
8. printChar(42.0)  → 42 >= 32 y <= 126 → char: '*'
9. printInt(42.0)   → int: 42
10. printFloat(42.0)  → float: 42.0f
11. printDouble(42.0) → double: 42.0
```

### Ejemplo 2: `./convert A`

```
1. isCharLiteral("A")  → false (length es 1, no 3)
2. isSpecial("A")      → false
3. isFloat("A")        → false
4. isDouble("A")       → false
5. isInt("A")          → false (strtol no puede convertir "A")
6. literal.length() == 1 → true  ✓
7. d = static_cast<double>('A') = 65.0
8. printChar(65.0)  → 65 >= 32 y <= 126 → char: 'A'
9. printInt(65.0)   → int: 65
10. printFloat(65.0)  → float: 65.0f
11. printDouble(65.0) → double: 65.0
```

### Ejemplo 3: `./convert nan`

```
1. isCharLiteral("nan") → false
2. isSpecial("nan")     → true  ✓
3. char: impossible
4. int: impossible
5. literal == "nan" → float: nanf
                    → double: nan
6. return  (sale sin pasar por printChar etc.)
```

### Ejemplo 4: `./convert 3.14`

```
1. isCharLiteral → false
2. isSpecial     → false
3. isFloat       → false (no termina en f)
4. isDouble("3.14") → true  ✓  (tiene punto y strtod lo convierte bien)
5. d = 3.14
6. printChar(3.14)  → 3.14 < 32 → Non displayable
7. printInt(3.14)   → int: 3   (trunca los decimales)
8. printFloat(3.14) → float: 3.14f
9. printDouble(3.14)→ double: 3.14
```

## Resumen visual del diseño

```
texto de entrada ("42")
        │
        ▼
  ┌─────────────────────────────┐
  │   Detectar tipo              │
  │   isCharLiteral → 'A'       │
  │   isSpecial     → nan/inf   │
  │   isFloat       → 3.14f     │
  │   isDouble      → 3.14      │
  │   isInt         → 42        │
  │   length == 1   → A         │
  └─────────────────────────────┘
        │
        ▼
  convertir a double (tipo intermedio universal)
        │
        ▼
  ┌────────────┬─────────────┬──────────────┬──────────────┐
  │ printChar  │  printInt   │  printFloat  │  printDouble │
  │ con rangos │ con rangos  │  + sufijo f  │  + .0 si    │
  │ ASCII      │ INT_MAX/MIN │  + .0 si    │  es entero  │
  │            │             │  es entero  │              │
  └────────────┴─────────────┴──────────────┴──────────────┘
```

La clave del diseño es usar `double` como tipo intermediario universal,
porque es el más grande de los cuatro y permite llegar a todos los demás
con `static_cast`.
